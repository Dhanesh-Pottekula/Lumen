# Workflow: Effects — atmosphere only when it means something

Effects are the most-abused tool. The rule is simple: **an effect must encode the subject matter, or it
doesn't belong.** Default to none.

## When an effect is legitimate

- **Flow / particles that represent real movement** — ions through a channel, water down a river, cars
  on a road, current in a wire, trade along a route. The particles *are* the thing moving.
- **Glow that marks a real state** — a heated element, an active site, an energized region — where the
  glow means "this is hot / active / charged," not "this is pretty."
- **A preset that matches the physical event** — `rain`/`snow` in a weather scene, `smoke`/`fire` in a
  combustion scene, `energy` for a field. The preset must match reality.

## When it's not

Confetti to celebrate, sparks for excitement, glow to make something pop, particles as background
texture — all decorative. Cut them. The visual reviewer rejects decorative effects.

## Craft

- **Direction and rate carry meaning.** If you show flow, its direction and density should match the
  real thing (fast where the current is fast).
- **Effects are temporary teaching marks.** Turn them off when the process they represent ends.
- **Don't stack them.** One meaningful effect per moment; more reads as noise.

## Pitfalls

- Any effect you can't justify as "this represents X in the subject" → remove it.
- Repeated/idle glow → decorative; cut.
- Effect left running after its process ends → hide it.

Domains that lean on this: **biology** (ion/substance flow), **physics/earth-science** (fields,
weather, combustion), **economics/history** (trade/traffic flow). Most lessons use **none**.
