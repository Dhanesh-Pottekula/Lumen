# Workflow: Comparison — let the difference do the teaching

The fastest way to make a point clear is to put its opposite next to it. Comparison is cheap and
powerful — but only if you control the variable.

## The one rule: change exactly one thing

Hold everything identical between the two sides except the variable you're teaching — same framing, same
scale, same style. If both sides differ in five ways, the viewer can't tell which difference matters.
Slow vs fast: identical objects, different speed. Healthy vs diseased: identical anatomy, one change.

## Craft

- **Use `split` or `comparison` composition** so the two get equal, balanced zones (`main-left` /
  `main-right`).
- **Mirror the structure.** Same objects, same labels, same positions on each side, so the eye compares
  like with like.
- **Point at the delta.** Use `attention`/`emphasize` on the *difference* — the thing that changed —
  not the whole side. The comparison exists to spotlight one contrast.
- **Reveal together or in turn.** Show both sides at once for a static contrast; reveal the second after
  the first (build-up) when the second is a consequence of a change.

## Reach for it when

The lesson is fundamentally "X vs Y," "before vs after," "with vs without," "then vs now." If the
insight is a *difference*, this is the workflow.

## Pitfalls

- Sides differ in many ways → isolate one variable.
- Different scales/framing left/right → viewer compares apples to oranges; match them.
- No emphasis on the delta → the point is left implicit; mark it.

Domains that lean on this: **all of them** — it's the most universal workflow.
